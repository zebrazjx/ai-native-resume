# Candidate Material Library

Use this optional library when the user wants to reuse the same resume across many JDs.

Recommended structure:

```text
candidate-materials/
├── self-profile.md
├── resume-base.md
└── projects/
    ├── project-a.md
    └── project-b.md
```

## Usage Rules

- Treat the library as source material, not final resume text.
- Select only the most JD-relevant projects for each tailored version.
- Explain selected and excluded projects in the match report.
- Reframe the same project differently by role without changing facts:
  - Product: problem discovery, solution definition, prioritization, delivery
  - Growth: funnel, activation, retention, experiments, conversion
  - Engineering: technical choices, architecture, reliability, deployment
  - Operations: process design, execution, stakeholder coordination, scale
  - Commercial: customer needs, revenue, pricing, GTM, conversion
- Preserve the original ownership level and verified metrics.
